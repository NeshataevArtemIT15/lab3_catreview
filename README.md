Условие задачи:
Необходимо написать программу «телеграф», которая:

Принимает от пользователя строку на русском языке.
Преобразует каждую букву в код азбуки Морзе.
Выводит результат в виде последовательности точек (.) и тире (-).
Буквы разделяются пробелами, слова — двойным пробелом.

Используй азбуку Морзе для русского алфавита (как на изображении). Если символ не поддерживается (например, цифры или знаки), его можно игнорировать или заменять на ?.

Требования к реализации:

Язык: C++
Кодстайл: строго в стиле Epic Games (Unreal Engine):
PascalCase для функций и методов
CamelCase для переменных
Префиксы типов (например: F, U, A, если уместно, но можно без фанатизма)
Чистая структура, читаемость
Минимум using namespace std
Разделение на файлы (обязательно):
Header.h — объявления (класс/функции, словарь Морзе)
Source.cpp — реализация логики
main.cpp — точка входа
Архитектура:
Создать класс, например FTelegraph
Внутри:
std::unordered_map<char, std::string> с кодами Морзе
Метод EncodeMessage(const std::string& Input)
Обработка регистра (привести к верхнему)
Поддержка пробелов между словами
Дополнительно:
Программа должна корректно компилироваться и запускаться
Добавить пример ввода/вывода в комментариях
Код должен быть полностью рабочим

Формат ответа:
Выведи код строго по файлам:
/Header.h
<код>
/Source.cpp
<код>
/main.cpp
<код>

А Grok написал полную чушь, при вводе строки выдавал неверный результат и пришлось чутьчуть заморочиться.
Вот что было исправлено:
1. Ошибки в кодах Морзе (Source.cpp)
Ф: было "..-." → стало "..-." (верно, но исправлен дублирующийся пробел в словаре)
Ъ: было "--.-." → стало "--.-" (Ъ и Щ имеют одинаковый код в русской азбуке Морзе)
Э: было "..-.." → стало "..-." (5 знаков было ошибочно, правильно 4)
3. Ввод на Windows (main.cpp)
Заменён ненадёжный std::wcin (который ломается при SetConsoleCP(CP_UTF8)) на чтение через std::cin + конвертацию MultiByteToWideChar.
